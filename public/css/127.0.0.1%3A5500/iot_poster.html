IoT-Based Smart Traffic Density Monitoring System with Emergency Vehicle Detection

Problem Introduction

Traffic congestion is a major issue due to increasing vehicles and fixed-time traffic signals. Traditional systems cannot handle emergency vehicles efficiently, leading to delays. There is a need for a smart system that monitors traffic dynamically and prioritizes emergency vehicles like ambulances.

Solution Proposed

The system uses ESP32 with multiple ultrasonic sensors for multi-lane traffic monitoring and a Raspberry Pi with camera for detecting emergency vehicles. The system dynamically adjusts signal timing and gives priority to ambulance lanes. Data is sent to IoT cloud for monitoring.

Working Steps

  1. Ultrasonic sensors detect vehicles in each lane
  2. ESP32 calculates traffic density
  3. Camera captures road images
  4. Raspberry Pi detects ambulance using image processing
  5. If ambulance detected → priority signal given
  6. Otherwise → smart timing based on traffic density
  7. Data sent to IoT dashboard

Circuit Connections

  • Ultrasonic sensors → ESP32 GPIO
  • LEDs → ESP32 GPIO with resistors
  • ESP32 → WiFi (Cloud)
  • Camera → Raspberry Pi
  • Raspberry Pi → Decision signal to ESP32

Main Functionality

  • Multi-lane traffic monitoring
  • Ambulance detection
  • Priority-based signal control
  • IoT dashboard visualization
  • Smart traffic management

Limitations

  • Camera accuracy depends on lighting
  • Prototype scale model
  • Requires internet
  • Basic image processing

Smart Logic

  • Ambulance → Immediate Green
  • High Traffic → Longer Green
  • Low Traffic → Short Green
